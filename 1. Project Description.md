
# Amazon Brazil Sales Data Analysis using SQL
---
# Overview
This project focuses on analyzing Amazon Brazil’s e-commerce dataset to identify customer purchasing patterns, sales trends, payment behaviors, and product performance.

The analysis was conducted using PostgreSQL by working with multiple interconnected tables including customers, orders, products, sellers, payments, and order items.

The project covers SQL concepts ranging from basic aggregation queries to advanced analytical techniques such as joins, CTEs, subqueries, window functions, ranking, and cumulative analysis.

---

# Problem Statement
Amazon India wants to leverage insights from Amazon Brazil’s e-commerce operations to better understand:

- Customer purchase behavior
- Payment preferences
- Product category performance
- Revenue trends
- Seasonal sales patterns
- Customer segmentation
- High-value customer identification

The goal is to use data-driven insights to improve customer experience, optimize business strategies, and identify growth opportunities in the Indian market.

---

# Key Insights

- Credit cards were the most frequently used payment method across all order value segments.
- Certain product categories showed very high price variation, indicating premium product concentration.
- Seasonal sales analysis revealed peak revenue periods during specific months.
- Loyal customers contributed significantly to overall revenue.
- Monthly sales trend analysis helped identify peak and low-performing periods in 2018.
- Window functions helped track cumulative product sales growth over time.
- Payment-wise month-over-month growth analysis highlighted changing customer payment preferences.

# How to Run This Project

## 1. Create Database

```sql
CREATE DATABASE amazon_analysis;
```

## 2. Create Schema

```sql
CREATE SCHEMA amazon_brazil;
```

## 3. Import CSV Files

Import all dataset CSV files into PostgreSQL tables using pgAdmin Import/Export tool.

## 4. Run SQL Queries

Execute queries from:

```bash
AmazonBrazilAnalysis.sql
```

## 5. Analyze Results

Review outputs for:
- Customer insights
- Revenue trends
- Product performance
- Payment analysis
